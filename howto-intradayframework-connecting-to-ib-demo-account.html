There have been questions for connecting the intraday framework to the IB demo that is accessible here:
https://www.interactivebrokers.com/en/index.php?f=1286
After  you click on “Individual demo”  and give the .jnlp file permission to execute on your local system, choose the classic layout. You will then see

Now click on “Edit” , “Global Configuration”, then go to “API – Settings”.  you will see

Two things will need to be changed on this screen.
1) Check “Enable ActiveX and Socket Clients” to allow API connections
2) Change the “Socket Port” value to the value you are using in INTRADAY PartA and PartB, i.e.5001.  In this example I have used 5000.
The result will look like this:

Click “OK” or “Apply” and now you will be able to connect the INTRADAY PartA and PartB to the IB demo.

IMPORTANT: It seems that the IB demo will, for some contracts, return a wrong “liquid time” value (=time, in your local timezone, that the contract is tradeable). It will return ‘CLOSED’ instead of an actual timevalue. This will cause the strategy to refuse trading, even if it is getting marketdata, since the  ‘CLOSED’ value indicates a closed exchange, so there is no point in placing oders. It is possible to *hack* the strategy so that it ignores this information, for demo purposes. However this is not recommended for  paper or live trading for obvious reasons.
